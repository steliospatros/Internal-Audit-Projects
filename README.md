# Internal Security Audit: Botium Toys

## 📌 Project Overview
This project involves a comprehensive internal security audit for Botium Toys. The primary goal was to assess the organization's existing security controls, identify vulnerabilities, and provide actionable recommendations to align with the **NIST Cybersecurity Framework (CSF)**.

## 🎯 Audit Objectives
* Review current user access permissions and software update policies.
* Identify gaps in the existing security infrastructure.
* Ensure compliance with internal security protocols.
* Provide a roadmap for risk mitigation and remediation.

## 🛡️ Frameworks & Standards
* **NIST CSF:** Used to identify, protect, detect, respond, and recover.
* **Least Privilege Principle:** Applied to user access review.

## 🔍 Scope & Methodology
The audit covered the following assets and procedures:
1. **User Access:** Inventory of active accounts and permission levels.
2. **System Maintenance:** Analysis of software update frequency and patch management.
3. **Physical Security:** Review of office access and hardware protection.

## 📊 Key Findings & Remediation
| Vulnerability | Risk Level | Recommended Action |
| :--- | :--- | :--- |
| Outdated OS on workstations | High | Implement automated patch management. |
| Over-privileged user accounts | Medium | Apply Role-Based Access Control (RBAC). |
| Lack of MFA | High | Enforce Multi-Factor Authentication for all users. |
